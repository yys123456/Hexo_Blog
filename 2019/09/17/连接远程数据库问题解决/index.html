连接远程数据库问题解决

发表于 2019-09-17 | 更新于 2019-09-18 | 分类于 问题解决

卸载并在服务器上安装mysql

删除mysql以及残留项

sudo apt-get remove mysql-*

dpkg -l |grep ^rc|awk ‘{print $2}’ |sudo xargs dpkg -P

安装mysql并检查运行状态

sudo apt-get install mysql-client mysql-server

sudo service mysql status

开启、关闭、重启mysql服务命令

sudo service mysql start

sudo service mysql restart

sudo service mysql stop

注意：若在卸载期间或下载期间出现

Ignoring file xxxxx in directory ‘/etc/apt/sources.list.d/‘

处理方式是

sudo rm /etc/apt/sources.list.d/xxxxx

配置mysql的root用户的密码（使用mysql -u root -p来登陆）

mysql -u root -p 或 mysql

use mysql;

update user set authentication_string=PASSWORD(“密码”) where user=’root’;

update user set plugin=”mysql_native_password”;

flush privileges;

quit;

/etc/init.d/mysql restart;

mysql -u root -p 密码;

开启3306端口允许其他主机访问

先查看是否允许接入

netstat -an|grep 3306

找到/etc/mysql/my.cnf文件，查看内容

到/etc/mysql/mysql.conf.d/下编辑，把

bind-address = 127.0.0.1注释掉

进行授权

输入sudo mysql -u root -p

输入密码之后，进入mysql

授权给所有连接

grant all privileges on . to ‘root’@’%’ identified by ‘root’;

权限生效

flush privileges;

重启mysql

/etc/init.d/mysql restart

或

sudo service mysql restart

已经可以远程连接。
